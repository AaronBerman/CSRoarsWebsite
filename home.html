Our Mission

Background

Repositories of information, known as knowledge bases, come in two forms: machine readable and human readable. Machine readable repositories are encoded for interpretation by machines and makes them unsuitable for research by human eyes. Human-readable repositories store documents and physical texts that can be accessed by humans. While human readable knowledge bases are more interactive they require time-consuming prompting to provide solutions. Knowledge management soulutions at present, mostly in corporate and government environments, are mostly collaborative tools or templates in which the user has to follow organizational policy on layout and filestorage to create a searchable repository. For machine repositories they are used for automating help desks.

Problem Description

Collection, storage, and processing of individualized data, in regards to electronic reference material, is fragmented and/or domain specific (such as finance and enterprise). Educators and researchers need a framework and tool-set to both collect and process data for the purpose of knowledge management.

Solution Description

Our goal is to build an extensible framework (ROARS) and a tool-set with which to collect and store electronic data in an organized structure, regardless of format or type. The framework can be extended throught he use of api calls. The base tool-set would include a webscraping utility, a utility to compare two versions of the same document, and a way to track changes to the document. The framework will allow for modules to be built for other types of processing of data after the initial storage and collection .
